958. Check Completeness of a Binary Tree                   A+
The code executes a Level-Order Traversal (BFS) where every node's children (including nullptr) are pushed into the queue.
The first loop processes valid nodes until it hits the very first nullptr at the front of the queue, which marks the potential 
end of the tree. The second loop then discards all subsequent nullptrs; if the queue is completely empty afterward,
it confirms that no actual nodes existed after the first gap (valid), whereas if any node remains, it means the tree had 
a gap and is incomplete.
