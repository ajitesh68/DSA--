If the final sum equals the target, we return 1 (a valid path), otherwise 0

We used a map because current_sum can become negative (e.g., 0 - 5 = -5). Standard vectors only

A map allows negative keys naturally, avoiding the need for extra math (index offsets) to shift 
negative sums into positive array indices.

1049. Last Stone Weight II                       B 

1. The Direct + / - Approach (Simulation)
Thought Process: "I want to simulate every possible game outcome." You treat the problem as assigning a sign (+ or -) to every 
stone. At each step i, you branch into two parallel universes: one where you add the stone to your current total, and one where 
you subtract it. When you reach the end of the array, the absolute value of your total represents the final stone weight for that 
specific sequence of choices. You compare all these final values and return the smallest one. Since the running total can dip 
below zero, you add an offset (like 3000) to map negative sums to valid array indices.

2. The Subset Sum Approach (Mathematical Optimization)
Thought Process: "I want to split the stones into two equal piles." You realize that minimizing the difference between Group A 
(positive stones) and Group B (negative stones) is mathematically the same as trying to make Group A sum up to exactly half of the 
total weight. Instead of tracking + and -, you only track Group A. You iterate through the stones and try to fill a knapsack with 
capacity TotalSum / 2. The closer you get to this half-way mark, the smaller the resulting difference will be. This eliminates 
negative numbers entirely and shrinks the table size.


1105. Filling Bookcase Shelves                                           B 
### Top-Down (Recursive) Thought Process                         

Think of this as making a decision for every book relative to the **active** shelf. At each index `i`, you have two choices: either 
**"Keep"** the book on the current shelf (if width permits) or **"Skip"** to close the current shelf (add its max height to the total) and 
start a fresh shelf with book `i`. We use a 2D memoization table `t[index][remaining_width]` because different paths often reach the same 
state of index and available space.

### Bottom-Up (Iterative) Thought Process                                

This approach solves the problem backward, from the last book to the first. `dp[i]` represents the minimum total height required for books 
from index `i` to the end. For each book `i`, we treat it as the **start** of a new shelf and use an inner loop to try placing subsequent 
books (`i`, `i+1`... `j`) on that same shelf until the width limit is hit. The value for `dp[i]` becomes the minimum of 
`current_shelf_height + dp[j+1]` across all valid split points `j`.

**1. Recursive Approach (Mental Model)**
We first identify this as a variation of **House Robber II** because we cannot pick adjacent items in a circle. Since circular dependencies are hard to handle directly, we break the problem into two linear scenarios: one ranging from index `0` to `N-2` (excluding the last) and one from `1` to `N-1` (excluding the first). The answer is the maximum of these two. In the recursion, we carry two states: `index` (current slice) and `count` (slices remaining to pick). At every step, we make a choice: either **take** the current slice (adding its value and skipping to `index + 2`) or **skip** it (moving to `index + 1`).

**2. Tabulation Approach (Structure)**
We translate the recursion into an iterative table to avoid stack overflow and overhead. We define a 2D array `dp[i][j]` representing the maximum sum possible considering slices up to index `i` while picking exactly `j` slices. We iterate through every index from `0` to `M` and every count from `1` to `limit`. 3154. Find Number of Ways to Reach the K-th Stair               B
We explore all possible paths using three state variables: current location (stair), current jump exponent (jump), and a flag to prevent 
consecutive backward steps. At each step, if stair == k, we count it (add 1) but continue recursing. From any state, we have two choices:

position values are too large for a standard array.
