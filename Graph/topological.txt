207. Course Schedule                           A->B->B

BFS Code Thought Process          ---------> Typical topological approach (Kahn's algorithm)
Build an adjacency list and count incoming prerequisites (indegree) for each course.
Push all courses with 0 prerequisites into a queue.
Process the queue: count the course as finished, reduce the prerequisite count for its dependent courses, and queue any that drop to 0.
If the total finished courses equal numCourses, no cycles exist, and the schedule is possible.

DFS approach finds cycles by tracking your current active traversal path.
* Mark the current node in both a general `visited` array and an active `pathVisited` array.
* Recursively visit its unvisited neighbors; if you encounter a neighbor that is already marked in `pathVisited`, you have hit a cycle.
* Once a node's exploration finishes, unmark it from `pathVisited` (backtrack) to prevent false cycle detections when exploring other 
independent paths.

802. Find Eventual Safe States                                        B -> A+    
BFS - kahn's algorithm
Is problem ka core intuition **"reverse thinking"** hai. Kahn's algorithm hamesha un nodes se start hota hai jinki *in-degree* 0 hoti hai,
lekin humein wo nodes chahiye jo safe end par terminate ho (yani jinki *out-degree* 0 ho). Toh agar hum graph ki saari edges ko **ulta 
(reverse)** kar dein, toh saare terminal nodes achanak se in-degree 0 wale starting nodes ban jayenge. Ab bas inhi naye starting nodes se 
Kahn's BFS chala do; cycle wale nodes aapas mein hi uljhe rahenge aur unki in-degree kabhi 0 nahi hogi, aur jo nodes queue mein aakar 
process ho jayenge wahi practically aapke "safe nodes" hain.

Cycle Elimination Nature: Kahn's algorithm ki sabse badi property hi ye hai ki wo cycles ko process nahi kar pata. Wo sirf cycle-free 
nodes ko queue mein lata hai. Safe nodes ki requirement bhi exactly yahi hai: jo cycle ka hissa na ho.

Dependency Chain: Question kehta hai ki "Ek node tabhi safe hai jab uske aage ke saare raste pehle se safe hon". Jab bhi aisi strict 
dependency ho (A depends on B), wahan hamesha Topological Sort fit baithta hai.

The "Reverse" Catch: Standard Kahn's in-degree (start) se chalta hai, par yahan dependency out-degree (terminal/end) par hai. Isiliye 
dimaag mein turant click karna chahiye ki agar graph reverse kar dein, toh end points starting points ban jayenge aur Kahn's algo 
automatically saare safe nodes ko chaant lega.

DFS approach                                                          A+ -> A
DFS approach ka core intuition sidha **"Cycle Avoidance"** hai. Ek node sirf tabhi safe mana jayega jab uske aage ka koi bhi rasta kisi 
cycle mein na phase. Hum standard DFS (visited aur pathVisited arrays ke sath) chalate hain yeh dekhne ke liye ki rasta kahan jaata hai. 
Agar chalte-chalte current path kisi aise node se takrata hai jo aage se wapas ghoom kar peeche aa raha hai (active `pathVisited`), toh 
wahan cycle hai aur wo unsafe hai. Lekin agar kisi node ki saari outgoing paths bina kisi cycle se takraye successfully khatam ho jayein, 
matlab wo pakka terminal (end) nodes par hi ruki hain, toh wahi strictly hamara "Safe State" hai.




310. Minimum Height Trees                                              B->B

**MHT Intuition & Triggers (For Quick Revision):**
* **Highest Degree $\neq$ Center:** Sabse zyada connections wala node jaruri nahi ki tree ka center ho. (Ek lambi line ke side mein jude 
   ek star node ko imagine karo). Minimum height tabhi milti hai jab root pure tree ke longest path ke bilkul beech mein (Centroid) ho.
* **The "1 or 2 Centers" Fact:** Graph theory ka universal rule hai ki kisi bhi tree ke exact center mein strictly ya toh `1` single node 
    hota hai, ya `2` adjacent nodes.
* **Onion Peeling Strategy:** Center seedha nahi milta, isliye bahar se andar ki taraf aana padta hai. BFS use karke sirf extreme ends 
    yani **Leaves** (jinki degree exactly `1` hoti hai) ko queue mein daalo.
* **Layer-by-Layer Trimming:** Har step mein bahari layer (leaves) ko tree se kaat do aur unke neighbors ki degree `-1` kardo. 
    Jaise hi kisi neighbor ki degree ghat kar `1` ho (matlab wo ab naya leaf ban gaya), use agle level ke liye queue mein daal do.
* **The Stopping Condition:** Ye trimming tab tak karni hai jab tak total bache hue nodes `≤ 2` na reh jayein. Jo last mein queue mein 
    bachenge, strictly wahi aapke exact tree centers honge.

329. Longest Increasing Path in a Matrix                                 
Indegree ka asli role: Indegree concept ensure karta hai ki ek node queue mein sirf ek baar push ho (jab uska indegree 0 ho jaye). Yeh ek synchronization barrier ki tarah kaam karta hai aur redundant 
overlap ko rokata hai.
Dono approaches ka core thought process simple hai:

### **1. DFS + Memoization (Top-Down)**                             dfs --> A+

Iska main idea hai: **"Mujhe nahi pata longest path kahan se shuru hoga, toh har cell se try karte hain."**

* **Recursion:** Har cell $(i, j)$ se uske neighbors mein jaao jo strictly bade (`>`) hain.
* **Base Case:** Agar kisi cell ke charo taraf koi bada element nahi hai, toh wahan se path length `1` hai.
* **Memoization:** Ek baar kisi cell ka result nikal gaya, toh use `dp[i][j]` mein save karlo taaki dubara calculate na karna pade.
* **Final Step:** Har cell ka result check karke maximum nikal lo.

---

### **2. BFS + Indegree (Bottom-Up)**                                bfs-> A

Iska main idea hai: **"Sabse chhote elements se shuru karke bade elements ki taraf layers mein badho."**

* **Indegree:** Har cell ke liye count karo ki uske kitne neighbors usse chhote (`<`) hain. Agar `indegree == 0`, matlab woh ek starting point (local minimum) ho sakta hai.
* **Initial Queue:** Saare `indegree == 0` wale cells ko queue mein daalo (yeh Level 1 hai).
* **Layer-by-Layer:** Queue se elements nikalo aur unke bade neighbors ki `indegree` kam karo. Jab kisi neighbor ki `indegree` zero ho jaye, use next level ke liye queue mein daal do.
* **Final Step:** Jitne total levels traverse honge, wahi aapka longest path hoga.

1203. Sort Items by Groups Respecting Dependencies               B 
* **Core Logic Trigger:** "Before/after" (dependencies) aur "ordering" dekh kar turant Topological Sort dimaag mein aana chahiye.
* **The Dilemma:** Agar sirf items ko sort kiya, toh same group wale elements alag-alag jagah bikhar jayenge.
* **The "Aha" Moment:** Two-level sorting karni padegi. Pehle pure Groups ka order nikalna hoga, phir har group ke andar Items ka order set karna hoga.
* **Handling the `-1` Trap:** Agar saare `-1` (no group) walon ko ek hi entity maan liya, toh wo sab aapas mein chipak jayenge jo ki galat hai. Isliye har `-1` ko ek naya, independent dummy
 group ID dena zaroori socha.
* **Building Group Dependencies:** Agar item `U` ko item `V` se pehle aana hai, aur dono alag groups mein hain, iska seedha matlab hai ki `Group(U)` ko `Group(V)` se pehle aana hi padega. 
Yahan se do alag graphs banane ka idea aaya.
* **Final Assembly:** Groups ka ek valid order nikal lo. Phir har sorted group ko uthao, aur uske andar sirf wahi items daalo jo already sorted items wali list mein usi group ke the.
* **Failure Check:** Agar groups ya items mein cycle ban gayi (mutually dependent), toh topo sort fail ho jayega, matlab solution possible nahi hai, return empty.

1857. Largest Color Value in a Directed Graph                         B 
Yahan is question ko approach karne ki step-by-step thought process hai:

* **Identify the Core Problem:** Directed graph mein max color frequency nikalni hai. Agar cycle hai toh infinite path banega, isliye sabse pehle cycle detect karna zaroori hai.
* **Choose the Algorithm:** Path traverse karne aur cycle detect karne ke liye **Topological Sort (Kahn's Algorithm BFS)** sabse best choice hai.
* **Decide What to Track (DP State):** Har node tak aane wale sabse best path mein 26 colors ki frequency store karni hai. Iske liye `dp[N][26]` ka 2D vector banayenge.
* **Initialize BFS:** Queue mein sirf un nodes ko daalo jinki `in-degree` exactly 0 hai (jahan se path shuru ho sakta hai).
* **Process the Node (Update Self):** Queue se `node` nikalne ke baad:
* Us `node` ke khud ke color ki frequency DP mein `+1` karo.
* Ek global `max_val` variable mein maximum frequency update karo.


* **State Transfer (Path Forwarding):** `node` ke saare neighbors (`v`) par jane se pehle:
* 0 se 25 tak loop chala kar, parent (`node`) ki color values child (`v`) mein transfer karo: `dp[v][i] = max(dp[v][i], dp[node][i])`.
* Neighbor ki `in-degree` kam karo aur agar 0 ho jaye toh queue mein daal do.


* **Cycle Check (Final Step):** Ek `count` variable se track karo ki kitne nodes process huye. Agar `count != N` hai, matlab graph mein cycle hai, toh seedha `-1` return kardo.
