INTUITION:-To find a cycle in an undirected graph, we use DFS to explore paths. The key is to find an edge that leads to an already 
visited node. However, in an undirected graph, going from node A to B means B can also go back to A. This isn't a cycle, it's just 
the path we just came from. To solve this, we pass a parent variable in our DFS. When we explore a node, we mark it as visited. Then, 
for all its neighbors, if a neighbor is visited and it is not our immediate parent, we have found a true cycle (a back-edge). If the 
neighbor is not visited, we continue the DFS from there, passing the current node as the new parent.
