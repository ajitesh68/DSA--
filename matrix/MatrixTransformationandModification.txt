1329. Sort the Matrix Diagonally                 A+
You recognized that every diagonal originates from the matrix boundaries, specifically the left column or the top row. To capture all of 
them, you used two loops: one traversing down the first column and another across the first row. For each starting cell found, you 
followed the diagonal path to collect the values into a list, sorted them, and then overwrote the original path with the sorted numbers, 
ensuring every diagonal was processed independently.

Better approach and more calculative 
Think of every diagonal as having a unique "ID number" calculated by row - col. Negative IDs belong to diagonals starting on the top row,
while positive IDs belong to those on the left column. Instead of writing two separate loops to check the top and left edges, we simply 
loop through every possible ID number from the lowest (top-right) to the highest (bottom-left). For each ID, we calculate the starting 
cell, collect all the numbers in that diagonal, sort them, and put them back in order. This merges two distinct tasks into one efficient 
process.

2946. Matrix Similarity After Cyclic Shifts                           A+
The intuition is to verify if the matrix remains invariant under the shift without actually modifying it. Since a cyclic shift moves 
every element k steps, the matrix only looks the same if every element mat[i][j] is identical to the element located k positions away 
in the direction of the shift. For even rows, we compare the current element with the one k steps to the right (circularly), and for
