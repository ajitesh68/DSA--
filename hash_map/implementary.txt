3146. Permutation Difference between Two Strings                   A(20min)
INTUITION:-My goal is to sum abs(s_index - t_index) for every character. To do this, I need to pair up the indices for each character. 
Instead of using three loops (one for s, one for t, one for the map), I can do it in two. First, I'll create a "lookup" table 
(an array or map) that stores just the s indices, like {'a': 0, 'b': 1, 'c': 2}. Then, I'll loop through t just once. In this second
 loop, I already have the t_index (it's the loop variable i), and I can instantly find the s_index from my lookup table. This lets me 
 calculate the difference and add it to the total sum in a single pass.

Array used for **speed**.
1.  **Direct Access (O(1)):** A `char` (like `'a'`) is internally just a number (its ASCII value). Using `pos[t[i]]` lets the code use
 that number as a direct index to "jump" instantly to the correct slot.
2.  **Faster than a Map:** A `map` has to do a search (O(log N)), and an `unordered_map` has to compute a hash. The array is the fastest 
possible way to create this "character-to-index" lookup table.

2295. Replace Elements in an Array                                 A()
The core thought process is to optimize the "search and replace" operation from linear time $O(N)$ to constant time $O(1)$ using a hash map.

* **Index Mapping:** First, it iterates through the array to build a map (`val_to_index`) that stores the current position of every number (`value -> index`).
* **Direct Access:** Instead of scanning the array to find the `old` value for every operation, it uses the map to instantly retrieve the index.
* **Synchronized Updates:** It updates the actual array at that index and simultaneously updates the map (removing the `old` value and adding the `new` value) to ensure future lookups remain accurate.



3121. Count the Number of Special Characters II                    A+()
The thought process focuses on validating the strict order requirement: "all lowercase instances must appear before any uppercase instances."

To check this efficiently, the code identifies two critical boundary indices for each character:
1.  **Last Lowercase Position:** It scans forward to store the *last* index where a lowercase letter appears.
2.  **First Uppercase Position:** It scans backward to store the *first* index where an uppercase letter appears.

Finally, it compares these two indices. If the **last** lowercase position is strictly less than the **first** uppercase position,
 the condition holds true for that character, and the count is incremented.
