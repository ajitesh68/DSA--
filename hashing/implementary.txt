2363. Merge Similar Items                                                                            
**Hash Map Approach (`unordered_map` + Sort)**                               A
This method prioritizes fast  lookups for merging. We first map the second list to allow instant access, then iterate through the first list to add weights for 
overlapping items, removing them from the map as we go to mark them as "processed." Finally, we append any remaining unique items from the map to our list and explicitly

**Ordered Map Approach (`std::map`)**                                        B
This method leverages the intrinsic properties of a Red-Black Tree (the underlying structure of `std::map`) to handle both aggregation and sorting simultaneously. 
We iterate through both input lists and add their weights to the map; the map automatically handles key existence (adding to an existing sum or creating a new entry). 
Because the map keeps keys sorted at all times, we simply traverse it once to transfer the final data into a vector, eliminating the need for a separate sorting step.
