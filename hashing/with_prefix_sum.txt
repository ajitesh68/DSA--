1590. Make Sum Divisible by P                                   B 
**Thought Process:**
* Sabse pehle `total_sum % p` (target) nikalte hain. Agar 0 hai, toh kuch remove nahi karna.
* Humara goal exact target remainder wala smallest subarray dhoondhna aur hatana hai.
* Sliding window yahan fail hoti hai kyunki remainders (`% p`) randomly badhte ya ghatte hain, isliye pointers ko move karne ka logic nahi ban pata.

**Approach & Math:**
* Prefix sum ka concept lagate hain: `(curr_prefix - prev_prefix) % p = target`.
* Isko search karne ke liye formula banaya: `needed = (curr_prefix - target + p) % p`.
* Har index par current sum ka remainder nikalte hain aur map mein `needed` remainder ka latest index dhoondh kar answer update karte hain.

**Key Learnings:**
* Remainder aur subarray ki problems mein **Prefix Sum + Hash Map** sabse solid approach hai.
* C++ mein negative modulo ko positive banane ke liye hamesha `+ p` add karna chahiye.
* Map mein large total sum store karne ke bajaye sirf `% p` store karne se Memory (MLE) aur Time Limit Exceeded (TLE) dono bach jaate hain.

523. Continuous Subarray Sum                             A 
**Core Logic:** Prefix Sum + Modulo Property.
* **Prefix Sum & Modulo:** Har step par array elements ka sum karo aur `k` se remainder nikaalo (`pre_sum % k`).
* **Math Trick:** Agar 2 alag indices par remainder *same* aata hai, toh unke beech ka subarray sum $k$ ka exact multiple hota hai.
* **HashMap ka Use:** Map mein `(remainder, index)` store karte hain.
* **Checking Condition:** Agar current remainder map mein pehle se hai, toh bus check karo ki subarray ki length >= 2 hai ya nahi (`i - map[remainder] > 1`). Agar hai, toh return `true`.
* **Map Update:** Agar remainder pehli baar aaya hai, toh use map mein daal do taaki aage use ho sake.
* **Base Case (`map[0] = -1`):** Yeh zaroori hai taaki agar array ke start se hi koi valid subarray ban raha ho, toh uski length sahi calculate ho sake.

* **The Math Rule:** Agar do numbers ka difference kisi number k se fully divisible hai, toh un dono numbers ko k se divide karne par remainder bilkul same aayega.
    * *Example:* Maan lo k = 6. Index i par sum 8 hai (remainder 2). Index j par sum 14 hai (remainder 2). Unka difference (14 - 8 = 6) exact k ka multiple nikla!
* **Prefix Sum ki jagah Remainder kyu?**
    * Agar hum sirf Prefix Sum store karte, toh har nayi value ke aane par humein purani saari values se difference nikal kar check karna padta ki woh $k, 2k, 3k...$ ka multiple hai ya nahi (jo $O(n^2)$ time leta).
    * Remainder store karne se problem sirf ek lookup mein badal gayi: *"Kya maine yeh remainder pehle dekha hai?"* Agar haan, toh beech ka subarray $k$ ka multiple hai. Yeh HashMap se $O(1)$ mein ho jata hai.

2949. Count Beautiful Substrings II                       B 
Bilkul, chaliye is logic ko step-by-step break karte hain:

* **Mukhya Shart:** Question ke mutabiq ek valid substring ke liye `vowels == consonants` aur `(vowels * consonants) % k == 0` hona chahiye.
* **Variable L:** Agar `vowels` aur `consonants` dono barabar hain, toh dono ko `L` maan lijiye. Ab equation banegi `(L * L) % k == 0`.
* **Number x:** Hum ek sabse chota number `x` nikalte hain jiske liye `(x * x) % k == 0` true ho. Yeh `x` humein bata dega ki mod cycle kab repeat hoga.
* **Target Condition:** Ek valid substring tabhi banegi jab uske andar ke total vowels `L`, humare is `x` ke multiple hon (`L % x == 0`).
* **Prefix Check:** Prefix approach mein, sirf `curr_vowel - curr_conso` ka purane index se match hona yeh batata hai ki `vowels == consonants` hai.
* **Final Pair:** Modulo shart ko bhi poora karne ke liye, `curr_vowel % x` ka purane index wale `vowel % x` se match hona zaroori hai. Isliye hum map mein `{difference, curr_vowel % x}` ka pair check karte hain.
