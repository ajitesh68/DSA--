30. Substring with Concatenation of All Words                 A(90 min)

INTUITION(MY Approach):-The intuition for this code is to brute-force every possible starting position. 
First, it calculates the total length of a valid substring (e.g., 3 words * 3 chars/word = 9 chars). 
Then, it iterates through the main string s, one character at a time. At each starting index i, it extracts a substring of 
that exact total length. This substring is passed to a check function, which acts as a verifier. The check function breaks the substring
into word-sized chunks and uses a frequency map to ensure it contains exactly the right words in the right amounts, returning true only
if it's a perfect permutation.

INTUITION(optimal approach witout TLE):-   The core thought process is to move from a "brute-force" check to an efficient "sliding window."
We first realize that checking every possible substring from scratch (the check function) is too slow, especially for large inputs, 
as it leads to a Time Limit Exceeded error. The key insight is that since all words are the same length (wordLen), we don't need to 
re-evaluate everything when we move one character. Instead, we can "slide" a window, one wordLen at a time. However, a valid match might
not start at index 0 (e.g., it could start at 1, 2, etc.). To solve this, we run the sliding window algorithm wordLen times, one for 
each possible starting offset (0, 1, ..., wordLen - 1). For each offset, we use two frequency maps: one (wordCount) to store the target
counts of all words we need, and another (seenWords) to count words in our current window. As we slide the window by adding a new word
on the right, we check: 
1) If the new word is invalid, we reset the window. 
2) If the new word makes our window invalid (e.g., too many "foo"s), we shrink the window from the left until it's valid again. 
3) If the window size is correct (it contains numWords), we've found a match, record the start index (left), and slide the window forward
   to find the next one.

209. Minimum Size Subarray Sum                              A+(15 min)
INTUITION(thought process):- The right pointer expands the window by moving forward and adding elements to a running sum.
         As soon as the sum is greater than or equal to the target, a valid window is found. The code records this window's length 
         (right - left + 1) and updates the overall mini length. It then shrinks the window from the left, subtracting nums[left]

         efficiently in a single pass
